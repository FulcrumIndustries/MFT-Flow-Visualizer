Help & Syntax Guide

Pattern Syntax

Define MFT flows using a simple text-based syntax. Enter one pattern per line.

SOURCE -> PROTOCOL DIRECTION -> MIDDLE -> PROTOCOL DIRECTION -> TARGET

Basic Examples

TESTA -> PeSIT PUSH -> ST -> SFTP PUSH -> TESTB TESTZ <- PeSIT PULL <- CFT -> SSH PUSH -> TESTF

Parallel Inputs (multiple protocols to same target)

Use (PROTO1 PUSH & PROTO2 PUSH) for parallel protocols:

A -> (JMS PUSH & FILE PUSH) -> SFC

Multiple Targets (branching)

Use (PROTO1 DIR -> T1 & PROTO2 DIR -> T2) to branch to multiple targets:

SFC -> (EBICS PULL -> Banque & FILE PUSH -> CFT)

Components

Entities: Names like TESTA, ST, CFT (alphanumeric + underscores + spaces)

Protocols: SFTP, PeSIT, HTTPS, FTP, AS2, SSH, JMS, EBICS, FILE, etc.

Directions: PULL (fetch data) or PUSH (send data)

Arrows: <- (left) and -> (right) indicate flow direction

Annotations

Each connection has editable annotation fields for pre-processing and post-processing notes. Click on the yellow dashed boxes to add annotations.
